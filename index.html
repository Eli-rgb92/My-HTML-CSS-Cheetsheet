HTML Basics

Document Structure

Element Description Example
<!DOCTYPE html> Must be the first line. Tells the browser this is an HTML5 document. Required for proper rendering and validation. Without it, browsers may use quirks mode. <!DOCTYPE html>
<html lang="en"> The root element that wraps all HTML content. The lang attribute specifies the page language for screen readers and search engines. Always include it for accessibility. <html lang="en">...</html>
<head> Container for metadata that isn't displayed on the page. Contains title, meta tags, links to CSS/JS files. Not visible to users but crucial for SEO and functionality. <head><title>Page</title></head>
<body> Contains all visible content shown to users - text, images, links, etc. Everything you want users to see goes inside the body tag. <body>Content</body>
<meta charset="UTF-8"> Defines character encoding (how text is interpreted). UTF-8 supports all characters and languages. Place in <head> within first 1024 bytes. Prevents garbled text. <meta charset="UTF-8">
<meta name="viewport"> Controls mobile rendering. Prevents mobile browsers from zooming out to show desktop width. Essential for responsive design. width=device-width makes content fit the screen. <meta name="viewport" content="width=device-width, initial-scale=1.0">

Text Elements

Element Description Example
<h1> - <h6> Hierarchical headings from most important (h1) to least (h6). Use ONE h1 per page for main title. Creates document outline for SEO and screen readers. Never skip levels (h1 to h3 is wrong). <h1>Main Title</h1>
<p> Creates a paragraph block with spacing above/below. Use for body text. Browsers add default margins. Groups related sentences together semantically. <p>Paragraph text</p>
<strong> Semantically marks text as important/urgent. Renders bold by default but also tells screen readers to emphasize. Use instead of <b> tag for better accessibility. <strong>Important</strong>
<em> Semantically emphasizes text (like stress in speech). Renders italic by default. Screen readers change tone. Use instead of <i> tag for better meaning. <em>Emphasized</em>
<br> Forces a line break without creating a new paragraph. Self-closing tag. Use sparingly - prefer <p> for paragraphs. Good for addresses, poetry, or single-line breaks. Line 1<br>Line 2
<code> Marks inline code snippets (like variable names, function calls). Renders in monospace font. Use for code within text. For code blocks, use <pre><code> together. <code>variable</code>
<pre> Preformatted text that preserves spaces, line breaks, and indentation. Usually combined with <code> for code blocks. Respects all whitespace - useful for code examples, ASCII art. <pre>Code block</pre>

Links & Media

Element Description Attributes
<a> Creates clickable hyperlinks to other pages, files, or sections. href attribute is required for navigation. Use target="_blank" for new tabs. rel="noopener" prevents security issues with external links. href, target, rel
<img> Embeds images. src is required (image URL/path). alt is required for accessibility - describes image to screen readers. width/height prevent layout shift. Self-closing tag. src, alt, width, height
<video> Embeds video players. src points to video file. controls adds play/pause buttons. autoplay starts automatically (usually blocked by browsers). Supports multiple formats with <source> tags. src, controls, autoplay
<audio> Embeds audio players for music, podcasts, sound effects. src points to audio file. controls adds play/pause. Similar structure to <video> but without visual display area. src, controls

Lists

Element Description Example
<ul> Creates an unordered (bulleted) list. Use for items without specific order (shopping lists, navigation menus). Each <li> becomes a bullet point. Can be styled with CSS to remove bullets or change appearance. <ul><li>Item</li></ul>
<ol> Creates an ordered (numbered) list. Use when sequence matters (instructions, rankings). Numbers automatically increment. Can change numbering style with CSS or type attribute. <ol><li>First</li></ol>
<li> Represents one item in a list. Must be direct child of <ul>, <ol>, or <menu>. Can contain other elements (links, images, nested lists). Browsers add default spacing. <li>List item</li>
<dl> Description list for term-definition pairs (like a dictionary). <dt> is the term, <dd> is the definition/description. Great for glossaries, metadata, FAQ sections. <dl><dt>Term</dt><dd>Description</dd></dl>

Semantic HTML

Element Description Purpose
<header> Introductory content at the top of a page or section. Often contains logo, site title, navigation. Can appear multiple times (page header, article header). Improves SEO and accessibility. Introductory content, site navigation
<nav> Marks navigation sections (menus, table of contents, pagination). Use for major navigation blocks, not every link group. Screen readers use this to skip to navigation. Helps with semantic structure. Navigation links
<main> Wraps the primary, unique content of the page. Should appear once per page (not in headers/footers). Helps screen readers jump to main content. All visible primary content goes here. Primary content of the page
<article> Independent, self-contained content that makes sense on its own (blog post, news article, comment, widget). Can be nested. Good for syndication/RSS feeds. Distinguishable content piece. Independent, self-contained content
<section> Thematic grouping of related content, usually with a heading. More generic than <article>. Use when content is part of a larger whole. Example: chapters, tab panels, grouped content areas. Thematic grouping of content
<aside> Content indirectly related to main content (sidebars, pull quotes, advertisements, related links). Can be styled separately. Screen readers may treat it as complementary information. Sidebar content, tangentially related
<footer> Footer information for nearest ancestor (page or section). Typically contains copyright, contact info, links, author credits. Can appear multiple times per page. Important for document structure. Footer information, copyright
<figure> Groups self-contained content (images, diagrams, code blocks) with optional caption. Content can be moved without affecting document flow. Often contains <img> or <code> with <figcaption>. Self-contained content (images, diagrams)
<figcaption> Provides caption/legend for <figure>. Should be first or last child of <figure>. Describes or explains the figure content. Improves accessibility and context for images/diagrams. Caption for figure element
<time> Represents time/date in machine-readable format. datetime attribute enables parsing by browsers/search engines. Useful for events, publication dates, schedules. Improves SEO and accessibility. Machine-readable date/time

CSS Basics

Selectors

Selector Description Example
element Targets all elements of that type. Low specificity. Use for base styles (body, p, h1). Example: p selects all paragraphs on the page. p { color: red; }
.class Targets elements with that class. Reusable - multiple elements can share a class. Use for styling groups of elements. Higher specificity than element selector. Most common selector type. .container { }
#id Targets unique element by ID. Highest specificity. Use sparingly - IDs should be unique per page. Good for one-off styling or JavaScript targeting. Avoid for styling (use class instead). #header { }
selector, selector Groups multiple selectors to apply same styles. Comma separates selectors. Use to avoid repeating styles. Example: applies styles to both h1 and h2 elements. h1, h2 { }
parent child Descendant selector - targets child elements at any nesting level. Space separates parent and child. Matches all descendants, not just direct children. Example: selects all paragraphs inside divs, regardless of nesting. div p { }
parent > child Direct child selector - only immediate children, not grandchildren. More specific than descendant. Use when you need precise targeting. > symbol required. Example: only direct list items, not nested ones. ul > li { }
element:hover Pseudo-class that applies styles when mouse hovers over element. Creates interactive feedback. Common for links, buttons. Use for better UX - shows elements are clickable. a:hover { }
element::before Pseudo-element that inserts content before element. Requires content property. Useful for icons, decorative elements, quotes. Can add content without HTML. ::before (double colon) is modern syntax. p::before { content: ""; }

Common Properties

Property Description Values
color Sets the text color (foreground). Use hex (#ff0000), rgb(), rgba(), or named colors. Contrast with background must meet WCAG standards for accessibility. Inherits by default. #hex, rgb(), named
background-color Sets the background color behind content. Doesn't affect border or margin. Use transparent to make transparent. Can layer with background-image. Important for readability. #hex, rgb(), transparent
font-family Defines which font to use. Provide fallbacks (comma-separated). Last should be generic (sans-serif, serif, monospace). Fonts without fallbacks may fail to load. Arial, sans-serif, serif
font-size Sets text size. px = fixed size, em = relative to parent, rem = relative to root (16px default), % = relative to parent. Use rem for responsive typography. 16px is readable base size. 16px, 1em, 100%, 1rem
font-weight Controls text thickness. normal (400) or bold (700) are most common. 100-900 range for variable fonts. bold makes text thicker, lighter values make thinner. Affects readability. normal, bold, 100-900
text-align Horizontally aligns text within element. left (default), center, right, justify. Applies to inline content. justify spreads words to fill width (can look awkward). left, center, right, justify
text-decoration Adds/removes text decoration lines. none removes underlines (common on links). underline, overline, line-through available. Can combine with color. Use for links, emphasis. none, underline, line-through
line-height Sets space between lines (leading). Unitless numbers (1.5) are best - multiply font-size. Affects readability. 1.5-1.6 is comfortable. Too tight = cramped, too loose = hard to read. 1.5, 24px, 150%
width Sets element width. px = fixed, % = relative to parent, auto = browser calculates. Default varies by display type. Can cause overflow if too large for container. 100px, 50%, auto
height Sets element height. Similar to width. Avoid fixed heights - let content determine (auto). vh = viewport height. Can cause scrolling issues if content overflows. 100px, 50vh, auto
margin Space outside element (between elements). Shorthand: top right bottom left. auto centers block elements. Negative values allowed. Collapses with adjacent margins. Creates visual breathing room. 10px, 10px 20px, auto
padding Space inside element (between border and content). Shorthand: top right bottom left. Background color applies to padding area. Creates internal spacing, makes content less cramped. 10px, 10px 20px
border Shorthand for border-width style color. Adds visible border around element. Common: 1px solid #000. Separates elements visually. Can be styled per side (border-top, etc.). 1px solid #000
border-radius Rounds element corners. 50% makes circle (on square element). px or % values. Modern look, softer appearance. Can specify each corner separately. Creates pill/card shapes. 5px, 50%
box-shadow Creates shadow behind element. Syntax: offset-x offset-y blur spread color. Creates depth, elevation effect. inset makes inner shadow. Multiple shadows possible (comma-separated). Modern, professional look. 2px 2px 4px rgba(0,0,0,0.2)
opacity Makes element transparent (0 = invisible, 1 = opaque). Affects entire element including children. Use rgba() for background-only transparency. Useful for overlays, hover effects, fading. 0.5, 1
display Controls how element renders. block = full width, new line. inline = wraps with content, no width/height. flex/grid = modern layouts. none = hidden, removed from flow. Fundamental property. block, inline, flex, grid
position Controls positioning method. static = normal flow. relative = offset from normal position. absolute = relative to nearest positioned ancestor. fixed = relative to viewport. sticky = hybrid scroll behavior. static, relative, absolute, fixed, sticky
z-index Controls stacking order (which element appears on top). Higher numbers = above. Only works on positioned elements (not static). Use for overlays, modals, dropdowns. Can create stacking contexts. 1, -1, auto

CSS Layout

Property Description Common Values
display Fundamental property controlling layout. block = stacks vertically, full width. inline = flows horizontally, content width. inline-block = inline with block features. flex/grid = modern layouts. none = completely hidden. block, inline, inline-block, flex, grid, none
position Changes positioning context. static = default, normal flow. relative = can offset from original position. absolute = positioned relative to nearest positioned parent. fixed = stays in viewport when scrolling. sticky = sticks when scrolling past. static, relative, absolute, fixed, sticky
top, right, bottom, left Offset positioned elements from their position context. top/right/bottom/left specify distance. Use with relative, absolute, fixed, sticky. Negative values allowed. auto = browser calculates. Creates precise positioning. 0, 10px, 50%, auto
float Legacy layout method - floats element left/right. Text wraps around. Was primary layout tool before flexbox/grid. Still useful for images in text, but avoid for page layouts. Creates float context. left, right, none
clear Prevents element from sitting next to floated elements. Forces element below floats. Use to "clear" floats - make element start on new line after floated content. both = clears both sides. Needed in old float layouts. left, right, both, none
overflow Controls what happens when content exceeds container size. visible = content shows outside (default). hidden = clips overflow. scroll = always shows scrollbar. auto = scrollbar only when needed. Prevents layout breaking. visible, hidden, scroll, auto
overflow-x, overflow-y Controls horizontal (x) or vertical (y) overflow separately. Useful when you want different behavior per axis. Example: horizontal scroll, vertical hidden. Prevents unwanted scrolling in one direction. hidden, scroll, auto

CSS Flexbox

Container Properties

Property Description Values
display: flex Creates a flexbox container. Children become flex items arranged along main axis. flex = block container, inline-flex = inline container. Modern, powerful layout method. Replaces float-based layouts. flex, inline-flex
flex-direction Sets main axis direction (how items flow). row = horizontal (left to right). column = vertical (top to bottom). -reverse = reverses order. Determines which axis is "main" vs "cross". Default: row. row, column, row-reverse, column-reverse
flex-wrap Controls whether items wrap to new line when they don't fit. nowrap = single line, items shrink (default). wrap = items wrap to new lines. wrap-reverse = wraps in reverse order. Prevents items from being squished. nowrap, wrap, wrap-reverse
justify-content Aligns items along the main axis (horizontal if row, vertical if column). flex-start = start of axis. flex-end = end. center = centered. space-between = equal space between items. space-around = space around each. space-evenly = equal space everywhere. flex-start, flex-end, center, space-between, space-around, space-evenly
align-items Aligns items along cross axis (perpendicular to main axis). stretch = fills cross axis (default). flex-start/end = start/end of cross axis. center = centered. baseline = aligned to text baseline. Great for vertical centering. stretch, flex-start, flex-end, center, baseline
align-content Aligns wrapped lines when flex-wrap allows multiple lines. Only works with wrap enabled. Controls spacing between lines. Similar values to justify-content but affects line spacing, not individual items. stretch is default. flex-start, flex-end, center, space-between, space-around, stretch
gap Creates consistent spacing between flex items. Single value = same gap all around. Two values = row column. Modern alternative to margins. Cleaner than managing individual item margins. Works in flexbox and grid. 10px, 1rem

Item Properties

Property Description Values
flex-grow Controls how much item grows to fill available space. 0 = don't grow (default). 1 = grow equally with others. 2 = grow twice as much. Proportionate growth. Items share extra space based on grow values. 0, 1, 2 (number)
flex-shrink Controls how much item shrinks when space is tight. 0 = don't shrink (stay size). 1 = shrink proportionally (default). Higher = shrink more. Prevents items from getting too small or maintains certain items' size. 0, 1, 2 (number)
flex-basis Sets initial size before growing/shrinking. auto = use width/height or content size. Specific size (200px) = that initial size. 0 = ignore item's content size. Starting point for flex calculations. auto, 200px, 0
flex Shorthand for flex-grow flex-shrink flex-basis. Most common: flex: 1 (grow equally, fill space). flex: 0 1 auto (don't grow, can shrink, auto size). Saves typing. Use this instead of individual properties. 1 1 auto, 0 1 200px
align-self Overrides container's align-items for this specific item. Allows individual items to align differently on cross axis. auto = use container's align-items. Same values as align-items but per-item control. auto, flex-start, flex-end, center, stretch
order Changes visual order without changing HTML. Lower numbers appear first. Default: 0. Negative numbers work. Useful for responsive reordering. Doesn't affect source order for screen readers. 0, -1, 1 (integer)

CSS Grid

Container Properties

Property Description Values
display: grid Creates a CSS Grid container for 2D layouts (rows and columns). grid = block container, inline-grid = inline. Powerful for complex layouts. Items automatically flow into grid cells. Most flexible layout system. grid, inline-grid
grid-template-columns Defines column structure and sizes. repeat(3, 1fr) = 3 equal columns. 200px 1fr = fixed column + flexible. auto = content-sized. fr = fraction units (share space). Can mix units. Creates column tracks. repeat(3, 1fr), 200px 1fr, auto
grid-template-rows Defines row structure and heights. Similar to columns but for vertical tracks. repeat(2, 100px) = 2 rows, 100px each. auto = content height. 1fr = equal share. Controls row sizing. Can have explicit or implicit rows. repeat(2, 100px), auto 1fr
grid-template-areas Creates named grid areas using ASCII art syntax. Each string = row, each word = column. "." = empty cell. Items use grid-area to place themselves. Visual way to define layout. Great for page templates. "header header" "main aside"
grid-gap / gap Creates space between grid items (not around edges). gap is modern shorthand. Two values = row-gap column-gap. Replaces margin hacks. Clean spacing between grid tracks. Works in flexbox too. 20px, 1rem 2rem
justify-items Aligns grid items horizontally within their cells (inline axis). start = left, end = right, center = centered, stretch = fills cell (default). Applies to all items in container. Overrides item's justify-self. start, end, center, stretch
align-items Aligns grid items vertically within their cells (block axis). start = top, end = bottom, center = centered, stretch = fills cell (default). Applies to all items. Controls vertical alignment in grid cells. start, end, center, stretch
justify-content Aligns the entire grid horizontally if grid is smaller than container (inline axis). start/end/center = aligns grid. space-between/around/evenly = distributes space. Only matters when grid doesn't fill container. start, end, center, space-between
align-content Aligns the entire grid vertically if grid is smaller than container (block axis). Similar to justify-content but for vertical axis. Controls grid's position within container when tracks don't fill space. start, end, center, space-between

Item Properties

Property Description Values
grid-column Places item across specific columns. 1 / 3 = from column 1 to 3 (spans 2 columns). span 2 = spans 2 columns from current position. 1 / -1 = entire row. Shorthand for grid-column-start and grid-column-end. 1 / 3, span 2, 1 / -1
grid-row Places item across specific rows. 1 / 3 = from row 1 to 3 (spans 2 rows). span 2 = spans 2 rows. Shorthand for grid-row-start and grid-row-end. Allows items to span multiple rows, creating complex layouts. 1 / 3, span 2
grid-area Shorthand for row-start / column-start / row-end / column-end. Or use named area from grid-template-areas. Single value = named area (header). Four values = row/column placement. Simplifies positioning. header, 1 / 2 / 3 / 4
justify-self Overrides container's justify-items for this item. Aligns item horizontally within its cell (inline axis). start = left, end = right, center = centered, stretch = fills width. Per-item control. start, end, center, stretch
align-self Overrides container's align-items for this item. Aligns item vertically within its cell (block axis). start = top, end = bottom, center = centered, stretch = fills height. Individual item control. start, end, center, stretch

Responsive Design

Feature Description Example
@media Applies styles conditionally based on device features (screen size, orientation, resolution). min-width: 768px = styles for screens 768px and wider. Enables responsive design. Breakpoints: mobile (320-767px), tablet (768-1023px), desktop (1024px+). @media (min-width: 768px) { }
viewport units Size relative to browser viewport. vw = viewport width (1vw = 1% of width). vh = viewport height (1vh = 1% of height). vmin = smaller of width/height. vmax = larger of width/height. Great for full-screen layouts, hero sections. vw (viewport width), vh (viewport height), vmin, vmax
rem Root em - relative to root element's font-size (usually 16px). 1rem = 16px (default). 2rem = 32px. Consistent across all elements regardless of parent. Best for typography. Change root font-size to scale everything. 1rem = root font-size
em Relative to parent element's font-size. 1em = parent's font-size. If parent is 20px, 1em = 20px. Compounds with nested elements. Good for component-level sizing. Use rem for global, em for local scaling. 1em = parent font-size
% Percentage relative to parent element's size. 50% width = half of parent's width. Context matters - width % is relative to parent width, height % to parent height. Great for fluid layouts, centering with transforms. width: 50%
min-width Sets minimum width element can shrink to. Prevents element from getting smaller than specified. Useful for readable text (min-width: 320px). Works with responsive layouts. Element can grow but won't shrink below this. min-width: 320px
max-width Sets maximum width element can grow to. Prevents overly wide content (max-width: 1200px for readable text). Element can shrink but won't exceed this. Common for containers on large screens. max-width: 1200px
clamp() Sets responsive value with min, preferred, and max. clamp(min, preferred, max). Preferred value responds to viewport (using vw). Never goes below min or above max. Perfect for fluid typography that stays readable. font-size: clamp(1rem, 2.5vw, 2rem)

HTML Forms

Element/Attribute Description Example
<form> Container for form inputs that get submitted together. action = URL to submit to. method = GET (in URL) or POST (in body). Required for form submission. Groups related inputs. Can validate on submit. <form action="/submit" method="post">
<input type="text"> Single-line text input for names, search, general text. name attribute is required for form submission. Can have placeholder, maxlength, pattern. Most common input type. Browsers may autocomplete. <input type="text" name="username">
<input type="email"> Email input with built-in validation. Mobile keyboards show @ symbol. Browsers validate email format on submit. Shows error if not valid email. Better UX than text input. Use for email addresses. <input type="email" required>
<input type="password"> Password input that hides text as user types (shows dots/circles). Use for sensitive data. Never display password value. Browsers may suggest password managers. Essential for login forms. <input type="password">
<input type="checkbox"> Checkbox for true/false selections. Can check multiple boxes. checked attribute = pre-checked. Use for optional selections, terms acceptance. Group with same name for arrays. Can be required. <input type="checkbox" checked>
<input type="radio"> Radio button for single choice from multiple options. Same name groups them - only one can be selected. Use for mutually exclusive choices (gender, payment method). checked = default selection. <input type="radio" name="choice">
<select> Dropdown menu for selecting from options. <option> elements are choices. User clicks to see options. Good for long lists. Can have optgroups. Use when space is limited or many options. Can be multiple-select. <select><option>Option</option></select>
<textarea> Multi-line text input for longer content (comments, messages, descriptions). rows/cols set initial size (can resize). No value attribute - text goes between tags. Better than input for paragraphs. Supports rows, cols, maxlength. <textarea rows="4" cols="50"></textarea>
<label> Associates text description with form input. for attribute links to input's id. Clicking label focuses input (accessibility). Required for screen readers. Improves UX - larger click target. Always use labels. <label for="name">Name</label>
<button> Clickable button for actions. type="submit" submits form. type="button" does nothing (needs JS). type="reset" clears form. Better than <input type="button"> - can contain HTML. Essential for forms. <button type="submit">Submit</button>
required Boolean attribute marking field as mandatory. Form won't submit if empty. Browser shows validation message. Improves UX - prevents incomplete submissions. Use for essential fields. Works with most input types. <input type="text" required>
placeholder Hint text shown when input is empty. Disappears when user types. Not a substitute for label - always include label. Use for examples ("Enter email"). Light gray text. Don't use for critical instructions. <input placeholder="Enter name">

Accessibility

Feature Description Example
alt attribute Required on all images. Describes image for screen readers and when image fails to load. Empty alt="" for decorative images. Descriptive text for informative images. Critical for accessibility and SEO. Always include meaningful alt text. <img src="photo.jpg" alt="Description">
aria-label Provides accessible name for element when visible text isn't sufficient. Screen readers announce this. Use for icon-only buttons, decorative elements that need context. Overrides element's text content for screen readers. <button aria-label="Close menu">X</button>
aria-labelledby References another element's id that labels this element. Useful when label is elsewhere in DOM. Connects label to element. Screen readers read referenced element's text. More flexible than aria-label. <div aria-labelledby="heading1">
aria-describedby References element that provides additional description/help text. Links error messages, hints, instructions to form inputs. Screen readers announce this after main label. Improves understanding of purpose/errors. <input aria-describedby="help-text">
role ARIA role attribute defines element's purpose to assistive technologies. Use when semantic HTML doesn't exist (div role="button"). Prefer native elements when possible. Helps screen readers understand element function. <div role="navigation">
tabindex Controls keyboard navigation order. 0 = in natural tab order (make focusable). -1 = focusable via JS but not tab. Positive numbers = specific order (avoid, creates confusion). Use 0 to make divs focusable, -1 to remove from tab order. <button tabindex="0">
lang Specifies language of element's content. Helps screen readers pronounce correctly. Improves SEO. Use on <html> for page language. Can use on elements for language changes. Required for accessibility. <html lang="en">
scope Defines table header scope (col, row, colgroup, rowgroup). Tells screen readers which cells header applies to. col = header for column, row = header for row. Essential for accessible data tables. <th scope="col">
semantic HTML Use meaningful HTML elements that describe content purpose (<nav>, <main>, <article>, <header>, <footer>). Better than generic divs. Improves SEO, accessibility, maintainability. Screen readers understand structure. <nav>, <main>, <article>
color contrast WCAG requires sufficient contrast between text and background for readability. 4.5:1 ratio for normal text, 3:1 for large text (18px+). Ensures visually impaired users can read. Use contrast checkers. Critical for accessibility compliance. 4.5:1 for text, 3:1 for large text
focus styles Visible indicators when element is focused via keyboard. Required for keyboard navigation. Use outline or border. Don't remove - customize instead. Helps users see where they are. Essential for accessibility - never set outline: none without replacement. a:focus { outline: 2px solid; }
